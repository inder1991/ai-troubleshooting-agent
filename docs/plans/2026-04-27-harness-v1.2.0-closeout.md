## Deferred to source-repo execution (S17 + companion)

The source-side `ai-harness` repo needs:

1. **Bump `HARNESS_CARD.yaml.version` 1.0.4 → 1.2.0** in the same
   commit that publishes the v1.2.0 tag. (In this consumer repo we
   reverted the bump in S13 because Q21 correctly flagged the drift —
   the consumer's pin is still v1.0.4 and will stay there until the
   source tag exists.)
2. **Run extract.sh** — its new B17 smoke-test (S14) gates the carve.
3. **Sign and push the tag**:
   ```bash
   bash tools/extraction/extract.sh
   cd /tmp/ai-harness
   git push --force origin main
   git tag -s v1.2.0 -m "ai-harness v1.2.0 — P1 hardening"
   git push origin v1.2.0
   gh release create v1.2.0 --notes-file RELEASES.md
   ```
4. **Per consumer**: bump `.harness-version` to `v1.2.0`, then bump
   `HARNESS_CARD.yaml.version` to `1.2.0` in the same commit (Q21
   enforces it), then run `python3 tools/sync_harness.py
   --trust-key 73A7AF8F04F40EC9`.
